# 城市管理系统产品需求文档（PRD）

## 1. 文档概述

### 1.1 产品定位
城市管理系统是一款面向城市管理执法人员的综合性管理平台，覆盖日常巡查、事件上报、任务分配、车辆管理、人员管理、垃圾分类检查等多个业务场景，实现城市管理工作的数字化、智能化。

### 1.2 用户角色
| 角色 | 描述 | 主要职能 |
|------|------|----------|
| 管理人员 | 城管团队/序化团队的队长等管理层 | 事件上报、任务分配、人员管理、事件验收 |
| 任务跟踪人员 | 一线执法人员、队员 | 日常巡查、任务执行、垃圾分类检查、随手拍上报 |

### 1.3 系统架构
系统分为两个主要入口：
- **后台管理端**：数据分析、任务管理、人员管理、车辆管理、基础数据维护
- **任务执行端**：日常检查、垃圾分类、随手拍、事件处理

### 2.4.5 事件整改表单字段

| 字段名称 | 字段类型 | 是否必填 | 交互规则 | 处理逻辑 | 异常逻辑 | 备注 |
|----------|----------|----------|----------|----------|----------|------|
| 整改负责人 | 人员选择器 | 是 | 仅显示任务跟踪人员 | 从 personType='任务跟踪人员'中选择 | - | - |
| 整改措施 | 文本域 | 是 | 最大 500 字符 | 描述具体整改方案 | - | - |
| 整改后照片 | 图片上传 | 是 | 至少 1 张 | 与整改前对比 | - | - |

### 2.4.6 事件验收表单字段

| 字段名称 | 字段类型 | 是否必填 | 交互规则 | 处理逻辑 | 异常逻辑 | 备注 |
|----------|----------|----------|----------|----------|----------|------|
| 验收人 | 人员选择器 | 是 | 仅显示管理人员 | 从 personType='管理人员'中选择 | - | - |

```
pending(待处理) → processing(整改中) → completed(已完成) → accepted(已验收)

```

### 2.4.9 异常逻辑
- 照片上传失败支持重试
- 提交失败保留表单数据
- 网络断开时显示离线提示
