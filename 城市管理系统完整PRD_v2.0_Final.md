## 1. 文档概述

### 1.1 产品定位
城市管理系统是一款面向城市管理执法人员的综合性管理平台，覆盖日常巡查、事件上报、任务分配、车辆管理、人员管理、垃圾分类检查、建筑垃圾清运管理等多个业务场景，实现城市管理工作的数字化、智能化。

### 1.2 用户角色
| 角色 | 描述 | 主要职能 |
|------|------|----------|
| 管理人员 | 城管团队/序化团队的队长等管理层 | 事件上报、任务分配、人员管理、事件验收、建筑垃圾审核 |
| 任务跟踪人员 | 一线执法人员、队员 | 日常巡查、任务执行、垃圾分类检查、随手拍上报、建筑垃圾清运 |

### 1.3 整体业务流程图
```mermaid
graph TD
    A[用户登录] --> B{选择入口}
    B -->|后台管理端| C[Dashboard 数据大屏]
    B -->|任务执行端| D[任务执行首页]
    
    C --> E[任务管理]
    C --> F[人员/车辆管理]
    C --> G[基础数据维护]
    C --> H[建筑垃圾审核]
    C --> I[AI 日报摘要]
    
    D --> J[我的任务/日常检查]
    D --> K[垃圾分类检查]
    D --> L[随手拍/事件上报]
    D --> M[建筑垃圾申请/核销]
    
    E --> N[创建/分配任务]
    N --> O[任务跟踪人员执行]
    O --> P[任务完成/签到签退]
    
    L --> Q[事件上报]
    Q --> R[整改处理]
    R --> S[事件验收]
    
    M --> T[提交清运申请]
    T --> H
    H -->|审核通过| U[生成二维码]
    U --> V[现场扫码核销]
```

| 指派人员 | 人员选择器 | 是 | 可选择所有人员 | 从所有在职人员中选择 | - | v2.0 迭代：不再限制仅管理人员 |

| 2 | 违挡 |

### 2.4.5 事件整改表单字段

| 字段名称 | 字段类型 | 是否必填 | 交互规则 | 处理逻辑 | 异常逻辑 | 备注 |
|----------|----------|----------|----------|----------|----------|------|
| 整改负责人 | 人员选择器 | 是 | 仅显示任务跟踪人员 | 从 personType='任务跟踪人员'中选择 | - | - |
| 整改措施 | 文本域 | 是 | 最大 500 字符 | 描述具体整改方案 | - | - |
| 整改后照片 | 图片上传 | 是 | 至少 1 张 | 与整改前对比 | - | - |

### 2.4.6 事件验收表单字段

| 字段名称 | 字段类型 | 是否必填 | 交互规则 | 处理逻辑 | 异常逻辑 | 备注 |
|----------|----------|----------|----------|----------|----------|------|
| 验收人 | 人员选择器 | 是 | 仅显示管理人员 | 从 personType='管理人员'中选择 | - | - |
| 验收意见 | 单选框 | 是 | 通过/驳回 | 选择驳回需填写原因 | - | - |
| 驳回原因 | 文本域 | 条件必填 | 验收意见=驳回时必填 | 最大 200 字符 | - | - |

### 2.4.7 事件状态流转
```
pending(待处理) → processing(整改中) → completed(已完成) → accepted(已验收)
                                      ↓
                              rejected(已驳回) → processing(重新整改)
```

### 2.4.8 交互规则
- Tab 切换通过 URL 参数控制，支持浏览器后退
- 事件列表支持下拉刷新
- 点击事件卡片进入详情页
- 支持按状态、时间、类型筛选
- v2.0 迭代：事件上报时可指派所有人员，不再仅限于管理人员

### 2.4.9 异常逻辑
- 照片上传失败支持重试
- 提交失败保留表单数据
- 网络断开时显示离线提示

### 2.4.10 事件上报流程图
```mermaid
graph TD
    A[进入事件上报] --> B{选择Tab}
    B -->|上报| C[填写事件信息]
    B -->|整改| D[选择待整改事件]
    B -->|验收| E[选择待验收事件]
    
    C --> F[上传现场照片]
    F --> G[提交上报]
    G --> H[状态变为 pending]
    
    D --> I[填写整改措施]
    I --> J[上传整改后照片]
    J --> K[提交整改]
    K --> L[状态变为 completed]
    
    E --> M{选择验收意见}
    M -->|通过| N[状态变为 accepted]
    M -->|驳回| O[填写驳回原因]
    O --> P[状态变为 rejected]
    P --> D
```

### 2.5.6 异常逻辑
- 手机号重复时提示"该手机号已存在"
- 删除最后一名管理员时阻止并提示
